#+TITLE: Tutorial
#+OPTIONS: toc:nil num:nil

In this tutorial you will encrypt a secret file into a git-safe vault and
restore it on a fresh clone. Here is the end result:

#+begin_src text
$ nimvault status
------------------------------------------------------------------------
  Vault status
------------------------------------------------------------------------
  [in-sync]   ~/.config/myapp/secrets.env
#+end_src

Your secrets are now committed as opaque GPG blobs. Clone on another machine,
run ~nimvault unseal~, and the plaintext files reappear.

* Install nimvault

#+begin_src bash
# Requires Nim >= 2.0
nimble install nimvault
#+end_src

Verify:

#+begin_src bash
nimvault --version
# nimvault 0.2.0
#+end_src

* Set up a test repo

#+begin_src bash
# Create a throwaway repo to practice with
mkdir /tmp/vault-demo && cd /tmp/vault-demo
git init
#+end_src

* Create a secret file

#+begin_src bash
# This file will be encrypted into the vault
mkdir -p ~/.config/myapp
echo "DB_PASSWORD=hunter2" > ~/.config/myapp/secrets.env
#+end_src

* Configure the vault recipient

#+begin_src bash
# Find your GPG key ID
gpg --list-keys --keyid-format long
# pub   rsa4096/9CCCE36402CB49A6 2024-01-01 [SC]
#                ^^^^^^^^^^^^^^^^ copy this

# Tell nimvault which key to encrypt to
mkdir -p .vault
echo "recipient = 9CCCE36402CB49A6" > .vault/config
# ^ replace with YOUR key ID
#+end_src

* Add the file to the vault

#+begin_src bash
nimvault add ~/.config/myapp/secrets.env
# Added:
#   id:   7f3a2b1c9e4d8f06
#   path: ~/.config/myapp/secrets.env
#   blob: .vault/7f3a2b1c9e4d8f06.gpg
#+end_src

The file path is stored with =~/= so it works on any machine. The ID is a
random 16-char hex string.

* Seal the vault

#+begin_src bash
nimvault seal
# Encrypting ~/.config/myapp/secrets.env
# Sealed 1 file(s).
#+end_src

* Commit

#+begin_src bash
git add .vault/
git commit -m "vault: add myapp secrets"
#+end_src

Only encrypted blobs and the encrypted manifest are committed. The original
filename (~secrets.env~) never appears in git history.

* Check status

#+begin_src bash
nimvault status
# [in-sync]   ~/.config/myapp/secrets.env
#+end_src

* Simulate a fresh checkout

#+begin_src bash
# Delete the plaintext to simulate a new machine
rm ~/.config/myapp/secrets.env

nimvault status
# [missing]   ~/.config/myapp/secrets.env

# Restore from the vault
nimvault unseal
# Unsealing ~/.config/myapp/secrets.env
# Unsealed 1 file(s).

# Verify the file is back (permissions: 600)
cat ~/.config/myapp/secrets.env
# DB_PASSWORD=hunter2

ls -la ~/.config/myapp/secrets.env
# -rw------- 1 user user ... secrets.env
#+end_src

* Clean up

#+begin_src bash
# Remove the demo repo and test file
rm -rf /tmp/vault-demo
rm ~/.config/myapp/secrets.env
rmdir ~/.config/myapp 2>/dev/null
#+end_src
